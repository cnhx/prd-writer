# Code of Conduct

## Our standard

PRD Writer is a small public project intended to be useful across multiple agent ecosystems.

We want contributions, discussions, and reviews to stay constructive, direct, and respectful.

That means:
- criticize ideas, not people
- prefer evidence over posturing
- keep feedback specific and actionable
- assume good intent, but do not excuse sloppy or harmful behavior
- avoid performative hostility, gatekeeping, or pile-ons

## Expected behavior

Examples of behavior that help this project:
- giving clear and concrete technical feedback
- reporting portability problems with reproducible details
- suggesting simpler designs instead of adding unnecessary complexity
- documenting tradeoffs instead of arguing from vibes
- respecting that contributors may use different tools, models, and workflows

## Unacceptable behavior

Examples of behavior that are not acceptable here:
- harassment, threats, or intimidation
- insulting, demeaning, or deliberately hostile language toward contributors
- sharing private information without permission
- intentionally derailing technical discussions
- submitting malicious examples, secrets, or internal company data
- treating one platform as the only valid way to use the project

## Scope

This Code of Conduct applies to:
- issues
- pull requests
- discussions
- release notes and contribution comments

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, pull requests, examples, or other contributions that violate this Code of Conduct.

If behavior becomes disruptive or harmful, maintainers may limit or revoke participation.

## Reporting

If you need to report a problem, open a private channel with the maintainer instead of escalating publicly.

If private reporting is not available, open a minimal public issue without exposing personal details.

## Common-sense rule

Do not make this project harder to use, less portable, less honest, or less safe.
